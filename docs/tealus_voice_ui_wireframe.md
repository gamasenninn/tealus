# tealus 音声UI ワイヤーフレーム

## コンセプト
**チャットは止めない、音声は上に重ねる**

- 画面遷移なし
- 下から出るボトムシート
- 音声は即送信
- 文字起こしとAI整形は遅延反映
- 必要なら文字起こしだけ編集可能

---

## 1. 通常状態

```text
┌──────────────────────────────┐
│ トーク履歴                     │
│                                │
│ こんにちは                     │
│ OKです                         │
│                                │
│                                │
├──────────────────────────────┤
│ [＋] [🎤] [メッセージ入力欄      ] [送信] │
└──────────────────────────────┘
```

ポイント:
- 🎤 が音声入力の入口
- テキスト入力と音声入力を並列に置く
- 既存のチャット体験を壊さない

---

## 2. 録音開始

```text
（下からスライドアップ）

┌──────────────────────────────┐
│ 🎤 録音中…                     │
│                                │
│   ● ● ● ● ●                  │
│                                │
│   00:03                        │
│                                │
│ [キャンセル]           [送信]   │
└──────────────────────────────┘
```

ポイント:
- 別画面ではなくボトムシート
- 波形は簡易表示でもよい
- 長押し開始 / 離して送信 でも成立

---

## 3. 送信直後

```text
┌──────────────────────────────┐
│ 🎤 音声メッセージ               │
│ ▶︎ 再生                        │
│ ⏳ 処理中…                     │
└──────────────────────────────┘
```

ポイント:
- 音声は即時送信・即時表示
- 「待たされている」のではなく「送れている」感覚を優先

---

## 4. 文字起こし中

```text
┌──────────────────────────────┐
│ 🎤 音声メッセージ               │
│ ▶︎ 再生                        │
│ ⏳ 文字起こし中…               │
└──────────────────────────────┘
```

---

## 5. AI整形中

```text
┌──────────────────────────────┐
│ 🎤 音声メッセージ               │
│ ▶︎ 再生                        │
│ ⏳ AIが文章を整えています…     │
└──────────────────────────────┘
```

---

## 6. 完了状態

```text
┌──────────────────────────────┐
│ 🎤 音声メッセージ               │
│ ▶︎ 再生                        │
│ 📝 敵が来たので、回復薬を飲みました。 │
│ [文字起こしを編集]              │
└──────────────────────────────┘
```

ポイント:
- 編集可能なのは文字起こしだけ
- 通常のテキストメッセージは編集不可
- 音声原本はそのまま残す

---

## 7. 編集モード

```text
┌──────────────────────────────┐
│ 🎤 音声メッセージ               │
│ ▶︎ 再生                        │
│                                │
│ [テキストエリア]               │
│ 敵が来たので、回復薬を飲みました。 │
│                                │
│ [キャンセル]         [確定]     │
└──────────────────────────────┘
```

ポイント:
- 編集対象はテキストのみ
- 音声データそのものは不変

---

## 8. 編集履歴

```text
┌──────────────────────────────┐
│ 文字起こし編集履歴              │
│                                │
│ v1: 敵がきた                   │
│ v2: 敵が来たので               │
│ v3: 敵が来たので回復した       │
│ v4: 敵が来たので、回復薬を飲みました。│
└──────────────────────────────┘
```

ポイント:
- 普段は最新だけ表示
- 必要なときだけ履歴を見られる
- 履歴があるなら複数回編集も可能

---

## 状態遷移

```text
recording
→ sending
→ transcribing
→ formatting
→ done
```

---

## 設計の要点

### 1. 音声は特別扱いでよい
- テキストは完成品
- 音声は未完成のまま先に送れる
- 後から文字になり、整い、必要なら直せる

### 2. 遅延は弱点ではなく進行中として見せる
- 「待ち」ではなく「成長中のメッセージ」として扱う

### 3. tealusの特徴になる
- まず話す
- あとで整う
- 最後に必要なら人が直す

---

## 一言コンセプト案

- **まず話そう。整えるのはあとでいい。**
- **未完成のまま送れるチャット。**
- **書く前に送れるメッセンジャー。**
