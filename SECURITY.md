# セキュリティポリシー

## 脆弱性の報告

Tealus に脆弱性を発見した場合、**公開 Issue を作成しないでください**。
代わりに、以下のいずれかの方法で非公開に報告してください。

### GitHub Security Advisories（推奨）

[このリポジトリの Security タブ](https://github.com/gamasenninn/tealus/security/advisories/new) から新しい Advisory を作成してください。
メンテナとの非公開のやり取りが可能です。

### メール

- 宛先: gamasenninn@gmail.com
- 件名: `[Tealus Security] <概要>`
- 本文: 再現手順、影響範囲、可能であれば修正案

## 対応方針

- 受領から **72 時間以内** に初回返信
- 深刻度に応じて優先対応（Critical / High は最優先）
- 修正版リリース前に報告者と調整（公表タイミング・クレジット含む）
- 修正後、CHANGELOG とセキュリティアドバイザリで公表

## サポート対象バージョン

現状は最新の main ブランチのみがサポート対象です。
v0.1.0 以降、リリースタグが切られた時点で以下のポリシーを採用予定:

| バージョン | サポート |
|-----------|---------|
| 最新マイナー | ✅ |
| 1 つ前 | ✅（重大な脆弱性のみ） |
| それ以前 | ❌ |

## 報告者への謝辞

責任ある開示を行っていただいた報告者は、本人の希望に応じて CHANGELOG およびリリースノートで謝辞を記載します。
